Interest-based website. Coffee

Project brief

Having made a webpage for the mobile view and a website for a desktop view, the last project of the basic web theme was a responsive interest-based website. I created a minimalist website about coffee suitable for different devices. The result was reached by implementing CSS media queries, grid and flexbox. I also learned backing up and improving design by conducting some user-orientated tests such as 5 second test.

Acquired competences

I learnt to set a structure to the workflow, advanced my knowledge in web design both compositional and multi-device convention-wise, got my hands-on vector graphics, created user-centred design by conducting tests and advancing in CSS by using CSS grid, flexbox and creating a responsive website with media queries.

Implemented design priniciples

To create a minimalist style of the website, firstly I took into consideration layout, most importantly the amount of negative space to create a clean and calm feeling, used the column grid system for the desktop and tablet views as well as implemented relevant Gestalt laws â€“ proximity and similarity. For all illustrations including logo, I decided to go with line art style which was a great addition to the minimalist style.

User-centred design

To make sure my design serves the needs of the user and makes it understandable to navigate, I conducted a 5 second test to find out whether there are any cosmetic changes to be applied. After finishing the project, I conducted the trunk test to see if the navigation and structure of the website is clear and usable.

Multi-device patterns and media queries

A very important part of the design management was multi-device pattern conventions that helped me to stay on track with UI and UX conventions. I used a few layout switching techniques to create the most suitable design for each query: a layout shifter was used to switch from desktop view to the tablet and mobile versions and the column drop was used to switch from tablet to mobile view.
